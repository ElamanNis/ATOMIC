QORGAU — Контроль и защита скважин

Описание: Система создана для решения конкретной боли — мониторинга давления и предотвращения утечек на удаленных скважинах, где нет стабильного покрытия 4G/LTE. Основной упор сделан на автономность и работу по радиоканалу LoRa.

Firmware (ESP32 + C++)
В коде реализован промышленный подход к стабильности:

Обработка прерываний (ISR): Датчик протечки `XKC-Y25` заведен на аппаратное прерывание. Это значит, что если трубу прорвет, клапан закроется мгновенно, не дожидаясь, пока основной цикл `loop()` дочитает остальные датчики.
Борьба с шумами: Датчики давления часто "плавают". В коде прописан усредняющий фильтр (фильтр скользящего среднего), чтобы на графике диспетчер видел ровную линию, а не "забор" из случайных пиков.
* Протокол данных: Все данные упаковываются в компактный JSON. Это удобно для парсинга на стороне сервера и позволяет легко добавлять новые датчики в будущем без переписывания всей структуры.

 Dashboard (Web)

Интерфейс написан на чистом JS + Chart.js для максимальной скорости работы.

Real-time: График обновляется динамически при поступлении пакетов из LoRa-шлюза.
Логирование: Каждое событие (открытие/закрытие клапана, скачок давления) записывается в системный журнал с меткой времени.

 Структура проекта

 /Firmware` — рабочие исходники для Arduino IDE (требуется библиотека ArduinoJson).
 /Dashboard` — фронтенд панели управления (открывается в любом браузере).
 /Schematics` — файлы проекта из EasyEDA (схема и разводка платы).

Почему это работает?

В отличие от готовых решений, эта система обошлась в ~24 000 тенге. При этом она решает главную задачу — исключает человеческий фактор. Если на объекте ЧП, система сначала купирует аварию (закрывает клапан), а потом "докладывает" об этом на пульт, а не наоборот.
